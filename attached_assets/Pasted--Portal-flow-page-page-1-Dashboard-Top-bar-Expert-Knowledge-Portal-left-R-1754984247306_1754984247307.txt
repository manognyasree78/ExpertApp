# Routing fix (no more 404)

**Define these exact routes** and use them in clicks:

```
/portal                 -> PortalShell (layout)
/portal/dashboard       -> PortalDashboard
/portal/contribute      -> PortalContribute
/portal/status          -> PortalStatus
/portal/earnings        -> PortalEarnings
/portal/support         -> PortalSupport
/portal/profile         -> PortalProfile   // open from avatar
/portal/*               -> redirect to /portal/dashboard
```

**Sidebar → path mapping**

* Dashboard → `/portal/dashboard`
* Contribute Knowledge → `/portal/contribute`
* My Knowledge Status → `/portal/status`
* Earnings → `/portal/earnings`
* Support → `/portal/support`
  *(remove Profile Settings from sidebar to avoid mis-routing)*

**Avatar menu (top-right)**

* Profile Settings → `/portal/profile`
* Logout → clear localStorage (`auth`, `onboarding`, `approval`) → go to landing or `/portal/dashboard` with defaults.

**Shell requirement:** `PortalShell` must render a **`<Outlet />`** so children pages load inside.
**Default:** visiting `/portal` should **redirect to \*\*\*\*\*\*\*\*`/portal/dashboard`**.
